PROBLEMAS IDENTIFICADOS:
========================

1. CRÍTICO - bg-gradient-primary no existe
   Problema: Clase CSS referenciada pero NO DEFINIDA
   Ubicación: tailwind.config.ts intenta usar var(--gradient-primary) que nunca se define en index.css
   Impacto: Hero no tiene color de fondo correcto
   Solución: Cambiar a bg-gray-900 (color sólido) O definir --gradient-primary

2. CRÍTICO - Viola requisito Notion (NO GRADIENTES)
   Problema: bg-gradient-primary implica usar linear-gradient
   Documento: CLAUDE.md prohibe gradientes
   Impacto: No es enterprise-grade Notion style
   Solución: Usar color sólido como Gray-900

3. ALTA - Overlay insuficiente (opacity-20)
   Problema: Muy bajo, imagen demasiado visible
   Impacto: Texto con bajo contraste, no cumple WCAG AA
   Solución: Cambiar a opacity-40 o agregar overlay adicional bg-black/40

4. ALTA - Dark mode completamente quebrado
   Problema: text-primary-foreground en dark mode es gris oscuro
   - Light: --primary-foreground es 0 0% 98% (blanco) ✓
   - Dark: --primary-foreground es 0 0% 9% (gris oscuro) ✗
   Impacto: Texto ilegible en dark mode
   Solución: Usar overlay oscuro dinámico que responda a dark mode

5. MEDIA - Botones outline hardcoded white
   Problema: border-white/20 y hover:bg-white/10 no responden a dark mode
   Solución: Usar border-current/30 o dark:border-white/20

6. MEDIA - Padding no responsive
   Problema: p-8 es demasiado en móvil
   Solución: p-4 sm:p-6 lg:p-8
